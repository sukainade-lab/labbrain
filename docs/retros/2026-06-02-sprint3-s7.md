## What still cost something

### 1. The recurring shared-DB CI flake (Kong 502) — surfaced across S6→S7, **fixed this sprint**
The live `integration` job intermittently 502'd ("invalid response from upstream" —
Supabase/Kong gateway) on otherwise-passing queries. It bit twice: S6's bridge gate
(CI pending) and **S7's ship** (a 502 on the *unrelated* S2 test `@AC-2.3`, cleared only
by a manual `gh run rerun --failed`). Because retro **P1** gates every ship on CI being
green, a transient flake = a blocked merge.
**Fixed this sprint** in PR #10: `vitest run --no-file-parallelism --retry=2` — a
transient blip self-heals (re-runs only the failed test, up to twice), a real regression
still fails all three attempts. Root-cause Kong readiness probe kept as backlog. The
fix's own CI run was green first-attempt. → captured as **process note P3** (already
implemented; encoded in `ci.yml`, sibling to L2).

### 2. QA −1: automated/CI-only verification — no manual 375px + axe walk cited
S7's QA −1 was the *same* note as S6's: verification was automated-only, with no manual
375px mobile walk and no axe-core accessibility pass cited as PR/score evidence. Both
score reports listed "capture a manual 375px + axe walk" as a deferred gap. Two data
points, identical cause → **new lesson L7** (added to `.claude/lessons.md`). This is the
`<bdi>` of this sprint: a predictable, mechanical miss worth encoding so it stops
costing the QA point.
