## What to improve — the UX focus

### Recurring pattern: the app-wide `#D97706` button fails AA, and every UI story re-litigates it
The **same root cause** dinged S8 and S9: a shared primary-button pattern (white-on-`#D97706`, 3.19:1) lives un-tokenized across ~12 files and fails WCAG-AA. Each UI story's L7 walk re-discovers it. The two stories handled it **differently**, and the difference is the whole lesson:

- **S8** found the contrast bug in the walk and **fixed it before `/5-eo-score`** → UX 9, **zero round-trips**.
- **S9** found the same class of failure on its own primary CTA but **deferred it** as "pre-existing / app-wide, not an S9 regression" → the scorer (correctly) dinged the primary CTA *of the story's surface* regardless of provenance → **UX 8 → one bridge round-trip** (88→90) to do the fix that S8 had done inline.

The scorer doesn't care whether an AA failure is inherited; it cares whether the story's own primary CTA ships accessible. "Pre-existing pattern" is not a shield when the offending element is *this story's* CTA. → **Propose lesson L9** (below).

The systemic fix is the spun-off **app-wide a11y follow-up**: extract the button into a shared AA-compliant token (amber-700 `#B45309`, white 5.02:1) so the remaining ~10 `#D97706` files stop re-triggering this on every future UI story. Until that lands, L9 keeps each story honest.

## Proposed L9 (for approval)

> ### L9 — Fix AA failures on the story's own primary CTA before scoring; "pre-existing" is not a shield
> **Trigger:** S8 and S9 hit the same app-wide button contrast failure (white-on-`#D97706` = 3.19:1, fails WCAG-AA). S8 fixed it inline before `/5-eo-score` → UX 9, no round-trip. S9 deferred it as "pre-existing / not an S9 regression" → the scorer dinged the primary CTA of the story's own surface anyway → UX 8 → a bridge round-trip to apply the fix S8 had done inline.
> **Rule:** when the L7 walk surfaces a serious/critical a11y failure (contrast, name, target size) on an element the story **ships or owns on its surface** — especially the primary CTA — fix it before `/5-eo-score`, even if the pattern is inherited app-wide. Provenance ("it was already like that") does not exempt the story's own primary control. If the proper fix is a shared token, do the token (or a scoped local fix) in the same PR; spin off only the *other* surfaces that the story doesn't touch.
> **Effect on scoring:** UX hat capped at 9 for any story that ships a known serious/critical a11y failure on its own primary CTA, regardless of whether the pattern pre-existed.
