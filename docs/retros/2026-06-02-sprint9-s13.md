## Headlines

1. **The story-mix prediction landed to the point.** Sprint 8's retro closed with a
   specific forecast: "S13 (versioning/re-index) is more domain/data than UI — expect
   it to plateau nearer 90 like the infra stories unless it ships meaningful UI." It
   plateaued at exactly 90. The 90↔94 swing across S10–S13 is now a well-characterized
   **story-selection signal, not a process drift**: mock-tested infra/seam/data stories
   (S10, S11, S13) cap at ~90 by their nature; net-new visible-surface UI stories (S12)
   reach 94. Four data points, one clean rule. Sprint planning can now *choose* the
   ceiling by choosing the story.

2. **The headline product guarantee was satisfied by construction, not by migration.**
   S13's whole risk was "replace a document + re-index without losing Q&A history."
   The decisive insight (carried from `/2-eo-dev-plan`) was that the schema *already*
   decouples them: `queries.citations` is a frozen `jsonb` snapshot with **no FK to
   `documents`**, so a replace never touches `queries`. The story collapsed from a
   feared history-migration into a *safe atomic re-index keeping the same `document_id`*
   — and the live L2 test proves the citation snapshot is byte-for-byte unchanged
   (`toEqual`) across a replace. Reading the data model before designing the change
   removed an entire class of work. Worth reinforcing as a habit, not a lesson.

3. **Atomicity isolated to one all-or-nothing seam.** The `replace_document_chunks`
   RPC is the single point where a document's chunks ever change — delete-old +
   insert-new in one tx, security-definer, tenant-guarded (`tenant mismatch` raise),
   `revoke public` / `grant service_role`. The destructive swap is deferred until the
   new chunks are parsed + embedded, so any pre-RPC failure (parse, embed-mismatch)
   leaves the prior revision **fully answerable** — the fail-safe (AC-13.5) is tested
   on all three failure paths. Pure composition of the S2 ingest + S12 replace-in-place
   seams; **zero new deps, no new env**. Same seam discipline that's held since S6.

4. **Zero lesson caps bit — fourth consecutive clean-hygiene ship.** Every applicable
   lesson was satisfied *proactively during coding/review*, none surfaced at scoring:
   L1 (route-seam every branch 401/401/404/403/400/400/413/200/200), L2 (live
   `describe.skipIf().sequential` unique-tenant + cleanup), L3 (CI green on PR #25
   head `ee07687` before score), L4 (replace control reachable + PUT caller wired),
   L5 (`<bdi>` on version digit + filename), L6 (no secret prefixes, no new env),
   L7 (live `npm run axe:walk` 0 violations @375px incl. the net-new `/documents`
   walk), L8 (AC-13.1…13.8 enumerated in `docs/stories/S13.md` pre-code), L9 (≥44px
   replace control, AA). The lessons are design-time defaults now, not score-gate
   catches — same posture as S11 and S12.
