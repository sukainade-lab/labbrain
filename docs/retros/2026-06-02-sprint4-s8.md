## Proposed L8 (for approval)

> ### L8 — Phase-2 backlog stories must enumerate their AC-N.N criteria before coding
> **Trigger:** S6 and S8 each lost a QA point because the BRD defines Phase-2 stories (S6–S16) as one-line backlog items, not enumerated `AC-N.N` acceptance criteria like S1–S5. AC traceability then rests on `@AC-N.N` **test-tag convention** with no BRD spec line to diff against — the scorer can't verify coverage is complete, only that tags exist.
> **Rule:** in `/2-eo-dev-plan` for any Phase-2 story, first write the story's `AC-N.N` acceptance criteria into the BRD (or a `docs/stories/SN.md` spec) as a checkable list, then code/tag against it. Tags must trace to an enumerated criterion, not invent the scope.
> **Effect on scoring:** QA hat capped at 9 for any story whose ACs exist only as a backlog one-liner with no enumerated `AC-N.N` spec.
