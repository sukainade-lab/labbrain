# S9 — Audit export: download Q&A log as PDF

> **Status spec (per lesson L8):** the BRD lists S9 only as a backlog one-liner
> (`brd.md:165`). This file enumerates the checkable `AC-9.N` acceptance criteria
> that `@AC-9.N` tests trace to. Written **before** `/3-eo-code`.

**Story:** As a lab's Quality/Technical Manager facing a JISM accreditation
assessment, I can download the lab's Q&A history as a PDF — proof that every answer
the lab relied on was source-traced to its own documents (document name + page).

**ICP:** owner/admin of an ISO 17025 lab (Amman), Arabic-native, faces a periodic
JISM assessment. The PDF is the audit-evidence artifact handed to the inspector.

## Architecture decisions (locked, `/2-eo-dev-plan story-9`)

- **Rendering:** server-side **headless Chromium** (`puppeteer-core` driving a
  system Chromium installed in the Contabo Docker image) renders a print-styled
  **RTL HTML template** → a deterministic A4 PDF. The browser engine is the only
  reliable Arabic shaper + bidi resolver; pure PDF libs are rejected for
  mixed-script compliance text. The Chromium call lives behind a
  `renderPdfFromHtml()` **seam** — mocked in all automated tests (CI never needs
  Chromium), real render verified in the manual walk.
- **Access:** **owner/admin only** (least privilege; audit evidence is privileged).
- **Scope:** optional inclusive `from`/`to` date range (JISM assessments are
  period-bounded); omitted → full history.
- **Residency:** all rendering happens in-house (Frankfurt/VPS). No third-party PDF
  service ever sees lab Q&A content.

## Acceptance criteria

### AC-9.1 — Access control + tenant isolation
- `GET /api/audit/export` requires an authenticated session → **401** otherwise.
- Caller role must be `owner` or `admin` → **403** for `member`.
- The exported rows come from the caller's **user-scoped** Supabase client, so RLS
  guarantees only the caller's `tenant_id` rows are ever included. A second tenant's
  queries must never appear (tested explicitly, sibling to AC-1.3/AC-2.4).

### AC-9.2 — Content completeness + fidelity
- Includes **every** `queries` row for the tenant in the selected range, ordered by
  `created_at` ascending (chronological audit order).
- Each entry renders: `question_text`, `answer_text`, `question_lang`,
  `found_answer` (a "غير موجود / not found" entry is visually distinct), the
  citation block `📄 [document_name] — صفحة [page_number]` per citation, the asking
  user's email, and `created_at` formatted **DD/MM/YYYY HH:MM**.

### AC-9.3 — Date-range filter
- Optional query params `from` and `to` (ISO `YYYY-MM-DD`), **inclusive**.
- Reversed (`from` > `to`) or malformed dates → **400** with a clear message.
- Omitted → entire history.
- The effective range (or "كامل السجل / full history") is echoed in the PDF header.

### AC-9.4 — PDF rendering (RTL, deterministic)
- Output is a valid A4 PDF (`%PDF` magic header), `dir="rtl"`, IBM Plex Arabic.
- Every dynamic mixed-script value (document names, ISO clause numbers, emails,
  digits, dates) is **bidi-isolated** (`<bdi>` / `unicode-bidi: isolate`) — L5.
- Header carries: lab name, generated-at timestamp, range, exported-by email.
- Empty log (no rows in range) → a **valid PDF stating "لا توجد سجلات"**, not an
  error response.

### AC-9.5 — Delivery + reachable entry point
- Response: `Content-Type: application/pdf` and
  `Content-Disposition: attachment; filename="labbrain-audit-<lab-slug>-<YYYYMMDD>.pdf"`.
- An owner/admin UI control ("تصدير سجل الأسئلة (PDF)" + date pickers) on the
  admin/dashboard page triggers the download. The control must be **wired and
  reachable** end-to-end (L4): UI → route → file.

### AC-9.6 — Export traceability (audit integrity)
- Each successful export is recorded in-residency (`audit_exports` row:
  `tenant_id`, `user_id`, `range_from`, `range_to`, `row_count`, `created_at`) with
  RLS, so the lab can show *who* produced *which* evidence *when*.
- The export log row is scoped to the tenant; never leaks across tenants.

## Out of scope (defer)
- Scheduled/emailed exports, CSV/XLSX formats, digital signatures, per-user export
  quotas. (Add as new stories if a paying lab asks.)
