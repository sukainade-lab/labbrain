### 🟡 QA capped at 9 by L7 — gated UI surfaces keep deferring the live axe walk

S10 ships a UI surface (the founder "النقل والامتثال (KSA)" migration tab), but the L7 verification was cited as **static** (computed AA contrast + `min-h-[44px]` audit + `<bdi>` audit + "reuses axe-cleared S8 chrome") — **not** a live 375px + axe-core walk on the new surface. This is a **structural** pattern, not a one-off lapse:
- **S8** (founder panel, platform-admin-gated): *did* a live axe walk → found a real 15-node contrast bug → fixed → QA 9 with the walk cited.
- **S9** (admin export, owner/admin-gated): *did* a live axe walk → found the export-CTA contrast failure.
- **S10** (founder migration tab, platform-admin-gated): the gated surface made a live walk expensive to stand up (seed a platform-admin session, render tenants with assorted `data_region`/`migration_status`), so it was **deferred to static** → QA cap.

The root cause is friction: **auth/role-gated surfaces make the L7 live walk costly enough that it gets skipped.** The fix isn't a new lesson (L7 already covers it and bit correctly) — it's **infrastructure**: a reproducible seeded-session axe-walk harness so the live walk is cheap enough to always run.
